# Telegram бот для расчёта нормы воды, калорий и трекинга активности

Бот, который помогает пользователю рассчитать дневные нормы воды и калорий, а также отслеживать тренировки и питание.

## Демонстрация работы
Чтобы не делать миллион скриншотов записал экран с демонстрацией работы бота и логов: https://youtu.be/oAIHYGh2YWk. Скрин, где видно логи об успешном деплое лежит в репо (successful_deployment.png)

## Команды

- `/set_profile` — настройка профиля
- `/profile` — показать профиль
- `/log_water <мл>` — записать воду
- `/log_food <продукт>` — записать еду
- `/log_workout <тип> <мин>` — записать тренировку
- `/check_progress` — прогресс по воде и калориям
- `/plot` — графики прогресса
- `/recommend` — рекомендации
- `/reset_day` — сброс дневных логов

## Примечания

- Данные хранятся в памяти (без базы данных) и сбрасываются при перезапуске.
- Погода берется из OpenWeatherMap и влияет на норму воды.
- Калорийность продуктов берется из OpenFoodFacts, при отсутствии можно ввести вручную.

## Деплой (на render.com)

### 1. Создание сервиса в Render
1) Перейдите в Render и нажмите **New +** → **Web Service**.
2) Подключите репозиторий и выберите ветку (например, `main`).
3) В поле **Environment** выберите **Docker**.
4) В разделе **Name** задайте имя сервиса
5) Нажмите **Create Background Worker**.

### 2. Переменные окружения
В настройках сервиса (**Environment**) добавьте:
- `TELEGRAM_BOT_TOKEN` — токен Telegram-бота
- `OWM_API_KEY` — ключ OpenWeatherMap
- `PYTHONUNBUFFERED=1` — чтобы логи сразу попадали в вывод

### 3. Запуск
1) Дождитесь завершения билда (Build logs должны быть зелёными).
2) Откройте вкладку **Logs** — должны появиться строки запуска бота.
3) Напишите боту в Telegram `/start` и убедитесь, что команды работают.
